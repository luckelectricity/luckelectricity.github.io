ES6 let

一、let和const

1.新增了let命令，用来声明变量，用法与var相同，但是其声明的变量只能在let所在的代码块内有效。

{
  let a = 10;
  var b = 1;
}

a // ReferenceError: a is not defined.
b // 1
let和var的区别
var a = [];
for (var i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6](); // 10

在上面这个函数中，i是var声明的，在全局中都有效，每次循环都会被覆盖，导致最后输出的是最后一次i的值，所以是10.

var a = [];
for (let i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6](); // 6

因为i是let声明的，i只在当前本次循环有效，所以每次循环的i都相当于一个新变量。

一句话理解闭包

javascript中的函数运行在它们被定义的作用域里，而不是它们被执行的作用域里。

我来解释下

用let的话，函数运行时因为let是块级作用域，所以函数运行时被定义的作用域是a[i]中的i，所以console.log(i);因为i是无法在外部作用域被访问到的，所以i就是当时保存函数的i值。

但是如果用var，就是全局作用域，所以无论a[i]中的i为什么，最后console.log(i)中的i在全局是存在的，为10，所以打印出来的就为10。

2.不存在变量提升

let和var不同，不存在变量提升的现象，所以要先声明，后使用，否则会报错
